# Bootstrap playbook

Route: **wavves-init** (`/wavves-init`)

```
- [ ] 1. Check whether <repo>/wavves/ exists. If it exists, read INDEX.md and
        AGENTS.md before writing. Adopt; do not overwrite.
- [ ] 2. Read the wavves-init skill (skills/wavves-init/SKILL.md) in full.
- [ ] 3. Run the wavves-init setup workflow. Create only missing owned files.
- [ ] 4. Return the file list and commit plan. Commit only when the operator
        explicitly asks or repo governance grants it.
- [ ] 5. Tell the operator the home is ready and the next step is `/charter`
        or `/wavves` plus a bounded task description.
```
